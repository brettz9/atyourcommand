Substitutions explained

Substitution sequences allow dynamic insertion of web content into a command line. They all begin and end with the delimiters "<" and ">", respectively. If you need these in raw form, add a backslash before the character (a single backslash before it will be stripped). If you need these delimiters anywhere within a resulting URL, you should be using %3C and %3E, respectively, anyways. A prefix or prefixes can be applied before any sequence.
save-temp
Added to indicate that the targeted contents should first be saved to a temporary file. This can be used for copying a whole file (or URLs too even if sending a PUT back to the server for the original location). Can be followed by whitespace and "overwrite=" and "yes", "no", or "prompt" to indicate what to do when a file in the location already exists. Can be followed by whitespace and "continue=" and "yes" or "no" for whether to continue execution when a file is found in the location and "overwrite" is "no" or "prompt" with a resulting "no" answer by the user. Can be followed by %N where "N" is the number of the directory argument below.
ucencode-
Encode the content for safe inclusion within a URI component (the portion beginning with "?")
uencode-
Encode the content for safe inclusion AS a whole URI
escquotes-
Escape quotes (Note that this might be applied automatically to content in the future)
Sequences:
eval
Evaluate the JavaScript following the "eval" and a space. Substitutions will be usable within the string.
pageURL
The URL of the current page
pageTitle
The title of the current page
pageHTML
The HTML source of the current page
bodyText
The text of the current page (the concatenated text of the current document's body content)
selectedHTML
The HTML source of the text highlighted on the current page
selectedText
The text highlighted on the current page
linkPageURL
The clicked link's location
linkPageURLAsNativePath
The clicked link's location as a native path (e.g., "C:\")
linkPageTitle
The fetched title of the clicked link
linkBodyText
The fetched body as text of the clicked link
linkPageHTML
The fetched HTML source of the clicked link
imageURL
The selected image's location
imageDataURL
The selected image content as a data: URL
imageDataBinary
The selected image content as a binary string
